# Jiuyu Machinery Global Site

## What We're Building

为 `jiuyumachinery.com` 规划一个面向全球访客的国际化机械品牌展示站，用于展示 Jiuyu Machinery 的公司形象、核心产品和行业专业度。站点首要目标是展示品牌与产品实力，同时兼顾承接来自 X 的流量，并为潜在客户提供清晰但不过度激进的询盘入口。

首版站点以前端展示为主，不包含复杂后台交易或在线下单。站点核心页面范围包括首页、产品详情页、公司介绍页、文章栏目和联系入口。产品素材主要参考并提取自 Alibaba 店铺：`https://jiuyujx.en.alibaba.com/?spm=a2700.micro_cgs_home.0.0.2b593e5flxjyMh`。站点整体视觉需要突出机械感、工业感和可信赖的重型设备气质，主题色偏深，风格参考 CAT 一类重工业品牌。

## Why This Approach

本次选择“品牌官网优先，产品目录辅助”的方向。首页重点承担品牌叙事、公司介绍、重点产品展示和信任建立，让来自 X 或搜索的访客先快速理解 Jiuyu Machinery 是谁、做什么、有什么实力。产品详情页再承接更丰富的产品信息，避免首页因 SKU 过多而变成普通外贸目录页。

这个方向比“纯产品目录站”更符合当前的主目标，也更容易建立国际访客对品牌的第一印象。同时保留独立文章栏目，用于承载机械行业知识、产品知识和应用场景内容，为后续内容传播和搜索流量打基础，但不要求在首页占据过重比重。

## Key Decisions

- 品牌定位：对外品牌名称暂定为 `Jiuyu Machinery`，域名使用 `jiuyumachinery.com`。
- 站点目标：三者都要，但优先级为“展示品牌与产品实力” > “获取询盘” > “承接 X 导流”。
- 语言策略：首发面向全球访客表达，采用英文 + 中文，并预留西班牙语、阿拉伯语及更多语言扩展能力。
- 站点形态：采用“品牌官网优先，产品目录辅助”的方案，而不是纯 catalog 站。
- 页面结构：首页展示公司与重点产品；产品详情页承接更多产品内容；单独设置介绍页面、文章栏目和联系入口。
- 文章定位：文章栏目独立为一个导航 tab，内容以产品知识、行业科普、使用场景为主；首页可选择展示最新文章，但不要求作为核心模块。
- 询盘策略：展示为主，询盘为辅；采用中等强度 CTA，在关键位置提供联系方式或询盘入口，但避免压过品牌展示。
- 视觉方向：整体采用深色、厚重、工业机械感风格，强调力量感、结构感和可信赖感，参考 CAT 的品牌气质。
- 素材来源：产品图片和基础产品信息优先从 Alibaba 店铺整理获取，再按站点叙事方式重组展示。
- 全球访问：站点必须部署在海外可稳定访问的托管平台，并通过全球 CDN 或边缘网络分发，确保海外访客可正常打开。
- 域名策略：`jiuyumachinery.com` 需要完成 DNS 托管、站点解析、HTTPS 绑定，并支持主域名与 `www` 的统一访问策略。
- 媒体策略：产品图片不能直接依赖 Alibaba 外链，需整理后由站点自身静态资源或独立对象存储/CDN 托管。
- 上线方式：代码可托管到 GitHub，并连接到适合前端静态站的部署平台，形成可持续更新的发布流程。

## Global Access and Launch Requirements

站点上线不仅要“能打开”，还要满足海外用户稳定访问、加载速度和品牌可信度的基本要求。首版应优先选择适合纯前端展示站的海外托管方案，避免自建复杂运维体系。代码仓库可放在 GitHub，网站部署可采用支持全球边缘分发和 HTTPS 的平台，例如 Vercel、Cloudflare Pages 或同类方案。

域名 `jiuyumachinery.com` 需要完成 DNS 配置，并与部署平台完成自定义域绑定。主域名和 `www` 都应可访问，并明确统一跳转策略。全站必须启用 HTTPS。为了保证图片稳定性和加载速度，产品图片需要从 Alibaba 来源整理后重新托管，不能长期直接依赖第三方热链。

需求层面需明确一条边界：技术方案和配置可由 AI 协助整理、生成和落地，但域名服务商、GitHub、部署平台、邮箱或消息工具等第三方账号的登录、授权和最终绑定，需要由站点所有者本人完成。

## Resolved Questions

- 需求文档统一放在 `docs/req/` 下，并使用 Markdown。
- 首版不是交易站，也不要求复杂后台能力，重点是高质量前端展示。
- 首页需要公司介绍和重点产品，而不是完整产品堆叠。
- 产品详情页可以扩展展示更多产品，形成“首页精选 + 内页展开”的结构。
- 文章需要独立栏目，不一定重压在首页，但首页可以展示最新文章。
- 除中文、英文之外，优先预留西班牙语和阿拉伯语扩展。
- 询盘主入口采用邮箱、WhatsApp 和联系表单并存。
- 产品分类不沿用 Alibaba 原始结构，官网需要重新整理一套更清晰的品牌分类体系。
- 站点需要支持海外稳定访问，因此必须补充域名解析、部署平台、HTTPS 和图片托管要求。
- 代码可以上传到 GitHub，并通过 GitHub 连接部署平台形成发布流程。
- AI 可以协助完成技术方案、文档、代码和配置准备，但第三方账号授权与域名控制操作需由站点所有者完成。
